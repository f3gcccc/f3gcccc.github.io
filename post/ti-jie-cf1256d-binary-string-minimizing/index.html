题解：CF1256D Binary String Minimizing

2024-09-25

贪心。

数据范围 n≤106n \le 10^{6}n≤106，因此我们要用时间复杂度为 O(n)\mathcal{O}\left( n \right)O(n) 的算法来解决这个问题。

思路

从左至右扫一遍序列，如果遇到 101010，则要将这个 000 交换到前面的位置。由于是字典序最小，000 应该尽量在最高位。现在需要知道这个 000 被交换到哪个位置，因此我们有变量 ttt 来记录下一个 000 所在的位置。

记刚才找到的 101010 的 000 的位置为 iii。则要交换 i−ti-ti−t 次，我们只需要判断 k≥i−tk\ge i-tk≥i−t 是否成立即可。

如果成立，则直接交换，将序列的第 ttt 个位置置为 000，第 iii 个位置，置为 111。这样成立的原因是因为序列的第 ttt 到 i−1i-1i−1 这些位置的数不可能有 000，因此我们并不需要去移动，直接 O(1)\mathcal{O}\left( 1 \right)O(1) 交换即可。

如果不成立，则尽量向前交换，将序列的第 i−ki-ki−k 个为位置置为 000（还能交换 kkk 次），第 iii 个位置置为 111，这样做的正确性与上文相同。

代码

注意小特判：如果序列的前面一段都是 000 则 ttt 的初始值应为这一段的最后一个位置 +1+1+1。

开 long long 否则爆炸，n2≥2147483647n^{2}\ge 2147483647n2≥2147483647。214748364721474836472147483647 为 int 最大值。

#include <bits/stdc++.h>
using namespace std;
#define int long long
int n,k,t;
const int maxn=1e6+5;
int a[maxn];

void Main(){
	cin>>n>>k;
	for(int i=1;i<=n;i++){
		char ch;
		cin>>ch;
		a[i]=ch-'0';
	}
	int t=1,g=1;
	while(a[g]==0) t++,g++;
	for(int i=2;i<=n;i++){
		if(a[i-1]==1&&a[i]==0){
			if(k>=i-t){
				k-=i-t;
				a[t]=0,a[i]=1;
				if(k==0) break;
				t++;
			}
			else{
				a[i-k]=0,a[i]=1;
				break;
			}
		}
	}
	for(int i=1;i<=n;i++) cout<<a[i];
	cout<<"\n";
}
signed main(){
	int T;
	cin>>T;
	while(T--) Main();
	return 0;
}
